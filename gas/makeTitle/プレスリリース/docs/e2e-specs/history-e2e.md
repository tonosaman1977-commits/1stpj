# 投稿履歴ページ E2E テスト仕様書

## テスト概要

| 項目 | 内容 |
|------|------|
| テスト対象 | 投稿履歴ページ（/history） |
| テスト種別 | E2E（エンドツーエンド） |
| 目的 | 投稿履歴の一覧表示・状態色分け・エラー表示・行クリックによるダイアログ表示・認証ガードが正しく動作することを確認する |
| 作成日 | 2026-03-07 |

---

## テスト前提条件

- テスト用アカウントが登録済みで、ログイン可能な状態であること
- テスト実行前に post_logs テーブルに既知のデータ（成功・失敗・pending 各パターン）が投入されていること
- 失敗レコードには error_message が設定されていること
- テスト環境のAPIが正常稼働していること
- ブラウザのJavaScriptが有効であること

---

## テストケース一覧

### 1. ページアクセス・認証ガード（ProtectedRoute）

| ID | テスト名 | 手順 | 期待結果 |
|----|----------|------|----------|
| TC-AUTH-01 | 未ログイン時のアクセス拒否 | 1. 未認証状態で /history に直接アクセスする | ログインページ（/login）へリダイレクトされ、投稿履歴ページは表示されない |
| TC-AUTH-02 | ログイン済みユーザーのアクセス許可 | 1. ログイン済み状態で /history にアクセスする | 投稿履歴ページが正常に表示される |
| TC-AUTH-03 | セッション切れ後のアクセス | 1. ログイン済み状態でセッションを無効化する（JWTを削除する等） 2. /history にアクセスする | ログインページへリダイレクトされる |

## 境界値・異常系テストケース

| ID | テスト名 | 手順 | 期待結果 |
|----|----------|------|----------|
| TC-EDGE-01 | 投稿内容がちょうど500文字のレコードの全文表示 | 1. content が500文字のレコードの行をクリックして Dialog を開く | Dialog 内に500文字全文が表示され、切り捨てや表示崩れが発生しない |
| TC-EDGE-02 | error_message が NULL のレコードのエラーアイコン表示 | 1. status=failed かつ error_message が NULL のレコードを確認する | ErrorOutlineIcon が表示される。Tooltip はエラーメッセージなしまたは「不明なエラー」等のフォールバックテキストを表示する |
| TC-EDGE-03 | theme_id が NULL（テーマ削除済み）のレコード表示 | 1. 紐づくテーマが削除済みのレコードの行を確認する | テーマカラムに「-」や「削除済み」等が表示され、エラーにならない |
| TC-EDGE-04 | 大量レコード（100件以上）のページ表示 | 1. post_logs に100件以上のレコードが存在する状態で /history にアクセスする | ページネーションまたはスクロールで全件アクセス可能で、ページの表示が崩れない |
| TC-EDGE-05 | 投稿内容に特殊文字（HTML タグ等）を含むレコードの表示 | 1. content に `<b>太字</b>` や `<script>` を含むレコードの行をクリックして Dialog を開く | スクリプトが実行されず、テキストとして安全に表示される |
| TC-EDGE-06 | retry_count が 2（最大再試行済み）の表示 | 1. retry_count=2 の失敗レコードの行を確認する | テーブルまたは Dialog 内に再試行回数が表示されている（または詳細確認可能な状態である） |
| TC-EDGE-07 | API エラー時のエラー表示 | 1. API が失敗を返す状態（ネットワーク切断等）で /history にアクセスする | エラーメッセージがUIに表示され、スタックトレース等の内部情報は表示されない |
| TC-EDGE-08 | ページリロード後に一覧が再取得される | 1. /history を表示した後、ブラウザをリロードする | リロード後も最新の投稿履歴一覧が表示される（キャッシュ問題なし） |
| TC-EDGE-09 | 別ユーザーの履歴 URL に直接アクセスできない | 1. ユーザーAでログインした状態で、ユーザーBの投稿IDを含む履歴詳細 URL に直接アクセスする | アクセスが拒否されるか、該当データが表示されない（403 または空表示） |
