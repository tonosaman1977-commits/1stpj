# 要件定義書 - 研究プレスリリース生成エンジン

## 1. プロジェクト概要

### 成果目標
研究概要テキスト（50〜20,000字）を入力するだけで、プレスリリース用6セクション（タイトル3案・リード文・背景・概要・意義・注意書き）を自動生成し、各セクションをコピペできるGASウェブアプリを提供する。

### 成功指標

#### 定量的指標
1. 入力から全セクション生成完了まで60秒以内
2. タイトル3案を含む全6セクションの生成成功率100%
3. 各セクションのクリップボードコピー成功率100%

#### 定性的指標
1. 研究者・広報担当者が専門的なライティングスキルなしにプレスリリース文案を得られる
2. 元研究にない事実追加・誇張なく、信頼性の高い文章が生成される
3. 生成後そのまま使える読みやすいレイアウトで、編集工数を大幅に削減できる

## 2. システム全体像

### 機能一覧

|--------|--------|------|

## 3. ページ詳細仕様

### P-001: プレスリリース生成画面

**目的**: 研究概要入力から6セクション生成・コピーまでを1画面で完結

**URL**: GAS WebApp デプロイURL

**処理フロー**:
```
1. ユーザーが研究概要テキストを入力
2. （任意）詳細設定を展開して補足情報を入力
3. 「生成する」ボタンクリック
4. クライアントバリデーション（50字以上・20,000字以下）
5. ローディングスピナー表示・ボタン無効化
6. google.script.run.generatePressRelease(params) 呼び出し
7. サーバー側バリデーション → Gemini API呼び出し
8. 6セクションをJSON構造で受信
9. セクションごとに表示
10. 各コピーボタンで navigator.clipboard.writeText() 実行
```

**入力仕様**:

| フィールド | 種別 | 制約 |
|-----------|------|------|
| researchText | 必須 | 50〜20,000字 |
| organizationName | 任意 | 大学名・研究機関名 |
| researcherName | 任意 | 研究者名・研究室名 |
| targetAudience | 任意 | 想定読者（一般/報道関係者/企業/教育関係者） |
| keywords | 任意 | 含めたいキーワード |
| constraints | 任意 | 避けたい表現・断定禁止事項 |

**出力仕様**:

| セクション | 内容 | 文字数目安 |
|-----------|------|-----------|
| titles | プレス向けタイトル3案（配列） | 各25〜50字 |
| lead | リード文 | 100〜180字 |
| background | 研究の背景 | 150〜300字 |
| overview | 研究内容の概要 | 200〜400字 |
| significance | 社会的意義・期待される活用 | 120〜250字 |
| notes | 注意書き（任意・不要時はnull） | - |

## 4. データ設計概要

### Gemini APIリクエスト構造
```json
{
  "contents": [{ "parts": [{ "text": "<構築済みプロンプト>" }] }],
  "generationConfig": {
    "responseMimeType": "application/json",
    "responseSchema": {
      "type": "OBJECT",
      "properties": {
        "titles": { "type": "ARRAY", "items": { "type": "STRING" } },
        "lead": { "type": "STRING" },
        "background": { "type": "STRING" },
        "overview": { "type": "STRING" },
        "significance": { "type": "STRING" },
        "notes": { "type": "STRING", "nullable": true }
      },
      "required": ["titles", "lead", "background", "overview", "significance"]
    }
  }
}
```

### バリデーション
- クライアント: researchText 50字以上・20,000字以下
- サーバー: 同上（二重チェック）

## 5. セキュリティ要件

- HTTPS強制（GAS WebApp はデフォルトHTTPS）
- 入力値はサーバー側でもバリデーション（文字数チェック）
- APIキーは PropertiesService のみ（コードへのハードコード禁止）
- ヘルスチェックエンドポイント: doGet パラメータ `?health=1` で `{"status":"ok"}` を返す

## 8. 生成ルール（プロンプト設計）

### 共通禁止事項
- 元研究内容にない事実の追加
- 誇張表現
- 効果・成果の断定

### セクション別ルール
| セクション | ルール |
|-----------|--------|
| titles | 3案生成・25〜50字・一般にも伝わる・煽らない |
| lead | 100〜180字・冒頭だけで研究概要が分かる |
| background | 150〜300字・なぜ必要か・従来課題・社会的背景 |
| overview | 200〜400字・何を・どう研究したか・一般読者向け |
| significance | 120〜250字・可能性として表現・断定しない |
| notes | 不要時はnull・検証段階・実用化保証なし等 |

### 任意入力の反映ルール
- organizationName: タイトル・リード文に「○○大学」として反映
- researcherName: リード文末尾に付記
- targetAudience: 語彙レベル・説明の詳細度を調整
- keywords: 各セクションに自然な形で含める
- constraints: プロンプト末尾に禁止事項として明示
