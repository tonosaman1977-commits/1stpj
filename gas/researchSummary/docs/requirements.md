# 要件定義書 - 研究要約エンジン

## 1. プロジェクト概要

### 成果目標
研究概要テキスト（20〜20,000文字）を入力すると、30秒説明・3分説明・研究要約の3種類をGemini APIで一括生成し、各セクションおよび全文をコピーできるGAS WebApp。

### 成功指標（定量）
1. 入力から生成完了まで30秒以内（Gemini API応答依存）
2. 3セクション（30秒・3分・要約）を1リクエストで一括生成
3. 入力上限20,000文字に対応（最低20文字）

### 成功指標（定性）
1. 研究者が場面（エレベーター・学内発表・研究紹介ページ）に合わせた説明文をゼロから作る時間をなくせる
2. 元研究にない事実追加・誇張なしのルールで安心して使える
3. ワンクリックコピーで即転用できる

## 3. ページ詳細仕様

### P-001: 研究要約生成ページ

**目的**: 研究概要を入力して3種類の説明文を生成・コピーする

**URL**: GAS WebApp エンドポイント（/）

**画面構成**
```
[研究要約エンジン]

研究概要を入力してください
[テキストエリア: 最低20文字、最大20,000文字]

[要約生成ボタン]  <- クリックでローディングスピナー表示

-----------------------------------------
30秒説明
[本文]
[コピー]

-----------------------------------------
3分説明
[本文]
[コピー]

-----------------------------------------
研究要約
[本文]
[コピー]

-----------------------------------------
[全まとめてコピー]

-----------------------------------------
[エラー時のみ表示]
エラーメッセージ
[リトライ]
```

**処理フロー**
1. 研究概要入力 → クライアントバリデーション（20文字以上）
2. 要約生成ボタンクリック → ローディングスピナー表示・ボタン無効化
3. `google.script.run.generateSummaries(text)` でGASサーバー呼び出し
4. GASサーバー → Gemini API（1回呼び出し、3セクション一括生成）
5. JSON応答をクライアントで受け取り → セクション別に表示
6. エラー時 → エラーメッセージ + リトライボタン表示

## 5. 出力仕様

### JSONスキーマ（Gemini responseSchema）
```json
{
  "type": "OBJECT",
  "properties": {
    "thirtySeconds": { "type": "STRING" },
    "threeMinutes":  { "type": "STRING" },
    "summary":       { "type": "STRING" }
  },
  "required": ["thirtySeconds", "threeMinutes", "summary"]
}
```

### セクション別仕様
| セクション | 表示名 | 長さ目安 | 用途 |
|-----------|--------|---------|------|
| thirtySeconds | 30秒説明 | 80〜120文字 | 研究紹介・イベント説明・メディアコメント |
| threeMinutes  | 3分説明  | 200〜350文字 | 学内発表・学部紹介・研究室紹介 |
| summary       | 研究要約 | 400〜600文字 | 研究紹介ページ・研究資料・学内共有 |

### 全まとめコピー形式
```
【30秒説明】
{thirtySeconds}

【3分説明】
{threeMinutes}

【研究要約】
{summary}
```

## 6. 生成ルール（プロンプト設計）

### 共通禁止事項
- 元研究内容にない事実の追加
- 誇張表現
- 効果・成果の断定

### セクション別ルール
| セクション | ルール |
|-----------|--------|
| 30秒説明 | 一文または短い段落・研究の核心を優先・エレベーターピッチ形式・80〜120文字 |
| 3分説明 | 背景・研究内容・意義を簡潔にまとめる・一般読者も理解できる・200〜350文字 |
| 研究要約 | 背景・方法・意義を構造的に整理・学術的すぎない・400〜600文字 |
